# AGENTS.md

이 저장소에서 작업하는 Codex 에이전트는 아래 지침을 따른다.

## 응답 언어와 정보 기준

- 기본 응답 언어는 한국어로 한다.
- 확실하지 않은 정보는 단정하지 않는다. 확인할 수 없으면 확인할 수 없다고 말한다.
- 최신 정보, 외부 서비스 동작, 라이브러리/API 변경처럼 변동 가능성이 있는 내용은 가능한 최신 근거를 확인한다.
- 저장소 안에 Knowledge 파일이 있으면 응답이나 작업 전에 먼저 확인한다. 현재 확인된 별도 Knowledge 파일이 없으면 없다고 판단하고 진행한다.
- 번역 요청은 요약하거나 의역하지 말고, 원문 의미와 형식을 가능한 그대로 유지한다.

## 프로젝트 맥락

- 이 프로젝트는 Streamlink와 FFmpeg를 사용하는 CHZZK 자동 녹화 프로그램이다.
- 사용자가 `chzzk_rekoda.py`라고 말하면, 저장소의 실제 메인 파일인 `chzzk_record.py`를 의미하는지 먼저 코드 구조에서 확인한다.
- `plugin/chzzk.py`는 CHZZK Streamlink 플러그인 관련 코드이고, `chzzk_record.py`는 녹화 실행 및 인코딩 옵션 구성의 중심 파일이다.
- 사용자가 불필요하다고 한 테스트 파일은 다시 만들지 않는다. 새 테스트는 명시적으로 요청되었거나 위험도가 큰 변경에 꼭 필요한 경우에만 추가한다.

## 작업 방식

- 코드를 수정하기 전에 관련 파일과 현재 git 상태를 확인한다.
- 검색은 우선 `rg` 또는 `rg --files`를 사용한다.
- 기존 코드 스타일과 구조를 우선 따른다.
- Python 코드는 PEP 8을 따르고, 불필요한 대규모 리팩터링은 하지 않는다.
- 수동 파일 수정은 가능하면 patch 방식으로 한다. 환경 제약으로 patch 도구가 실패하면, 최소 범위의 안전한 대체 방법으로 수정하고 그 사실을 사용자에게 알린다.
- 사용자가 요청하지 않은 원격 push, PR 생성, destructive git 명령은 하지 않는다.

## 검증 기준

- Python 파일을 수정한 뒤에는 최소한 문법 컴파일을 확인한다.
- 가능한 경우 `git diff --check`로 whitespace 오류를 확인한다.
- FFmpeg 인코더 관련 변경은 실제 FFmpeg probe 또는 동등한 최소 실행 검증으로 확인한다.
- CHZZK/Streamlink 플러그인 변경은 현재 설치된 Streamlink API와 맞는지 코드 수준에서 확인한다.
- 검증하지 못한 항목은 최종 응답에 명확히 남긴다.

## 커밋 규칙

- 사용자가 커밋을 요청하면 관련 없는 변경과 섞지 않고 파일/논리 단위로 분리한다.
- 커밋 메시지는 특별한 요청이 없으면 영어로 작성하고, 제목뿐 아니라 본문에도 변경 이유와 영향 범위를 상세히 적는다.
- 사용자가 Codex가 커밋한 것으로 남기길 원하면 author와 committer를 모두 `Codex <codex@openai.com>`로 지정한다.
- 커밋 후에는 최근 커밋과 작업 트리 상태를 확인한다.

## 이번 세션에서 이어받을 중요한 관례

- AV1/HEVC 인코더 문제는 FFmpeg에 인코더가 표시되어도 실제 하드웨어나 드라이버 문제로 런타임 실패할 수 있다는 전제로 다룬다.
- 하드웨어 인코더는 가능 여부를 probe하고, 실패하면 소프트웨어 인코더 fallback을 우선 고려한다.
- FFmpeg 실패와 Streamlink 실패 로그를 구분해 원인을 잘못 안내하지 않도록 주의한다.
- 빈 임시 녹화 파일을 정상 녹화 결과처럼 저장하지 않도록 확인한다.
